# プロジェクト紹介

このプロジェクトは、Next.js、Expo、Prisma、Better Auth を組み合わせた **モダンなフルスタック Turborepo モノレポ** です。Web アプリ、管理ダッシュボード、ドキュメントサイト、モバイルアプリを統合的に管理します。

## 🎯 プロジェクトの目的

- **統一された開発環境**: Web とモバイルを単一リポジトリで管理
- **共通コンポーネントの再利用**: UI ライブラリやユーティリティの共有
- **効率的なビルドとテスト**: Turborepo による並列実行とキャッシング
- **一貫した品質**: Lint、フォーマット、型チェックの統一ルール

## 📦 技術スタック

### フロントエンド

- **Next.js 15**: App Router による高速な Web アプリケーション
- **React 19**: 最新の React 機能とパフォーマンス改善
- **Tailwind CSS 4**: ユーティリティファーストの CSS フレームワーク
- **shadcn/ui**: カスタマイズ可能な UI コンポーネント

### バックエンド

- **Next.js API Routes**: サーバーレス API エンドポイント
- **Apollo Server**: GraphQL API サーバー
- **Prisma**: 型安全な ORM
- **libSQL (Turso)**: 軽量で高速な SQLite ベースのデータベース
- **Better Auth**: シンプルで強力な認証システム

### モバイル

- **Expo 54**: React Native の開発プラットフォーム
- **Expo Router**: ファイルベースのナビゲーション
- **Apollo Client**: GraphQL クライアント
- **Maestro**: E2E テストフレームワーク

### 開発ツール

- **Turborepo**: モノレポビルドシステム
- **pnpm**: 高速なパッケージマネージャー
- **TypeScript**: 型安全な開発
- **Vitest**: 高速なユニットテストフレームワーク
- **Playwright**: E2E テストフレームワーク（Web）
- **Ultracite (Biome)**: 高速な Lint・フォーマッター

### データフロー

1. **認証**: Better Auth が Web/Mobile の認証を統一管理
2. **API**: Backend が GraphQL/REST API を提供
3. **データベース**: Prisma が libSQL（Turso）または PostgreSQL（Supabase）にアクセス
4. **ストレージ**: Vercel Blob が本番環境のファイルストレージを提供

## 🌟 主な機能

### 認証システム

- メール + パスワード認証
- セッション管理
- 管理者承認フロー（オプション）
- セキュアなトークン管理

### API

- **GraphQL API**: Apollo Server によるクエリとミューテーション
- **REST API**: 標準的な HTTP エンドポイント
- 型安全なスキーマ定義

### データベース

- **Prisma ORM**: 型安全なデータアクセス
- **マイグレーション**: バージョン管理されたスキーマ変更
- **シード**: 開発・テスト用の初期データ
- **クラウド対応**: Turso Cloud / Supabase

### ファイルストレージ

- **環境適応型**: ローカル/本番で自動切り替え
- **Vercel Blob**: 本番環境の CDN 配信
- **ローカルストレージ**: 開発環境のファイルシステム

## 🎨 デザインシステム

### UI コンポーネント（@repo/ui）

- **50+ コンポーネント**: ボタン、フォーム、ダイアログ、テーブルなど
- **shadcn/ui ベース**: カスタマイズ可能なデザイン
- **Radix UI**: アクセシブルなプリミティブコンポーネント
- **ダークモード対応**: テーマの自動切り替え

### スタイリング

- **Tailwind CSS 4**: ユーティリティクラス
- **CSS Variables**: カスタムテーマ
- **Responsive Design**: モバイルファースト

## 🧪 テスト戦略

### ユニットテスト

- **Vitest**: 高速なテスト実行
- **React Testing Library**: コンポーネントテスト
- **カバレッジ**: コードカバレッジ測定

### E2E テスト

- **Playwright**: Web アプリの E2E テスト
- **Maestro**: モバイルアプリの E2E テスト
- **自動化**: CI/CD パイプラインで実行

## 🚀 デプロイメント

### Web（Vercel）

- **Preview**: Pull Request ごとのプレビュー環境
- **Staging**: `staging` ブランチのテスト環境（Pro 以上）
- **Production**: `main` ブランチの本番環境

### Mobile（EAS）

- **Development Build**: 開発用ビルド
- **Internal**: 社内テスト用配布
- **Production**: App Store / Google Play リリース
- **OTA Update**: Over-The-Air アップデート

### データベース（Turso Cloud）

- **環境別 DB**: Preview / Staging / Production
- **マイグレーション**: Prisma Migrate による管理
- **バックアップ**: 自動バックアップ

## 📊 開発ワークフロー

1. **ローカル開発**: `pnpm dev` で全アプリを起動
2. **Lint・フォーマット**: `pnpm format` で自動修正
3. **型チェック**: `pnpm check-types` で型エラー検出
4. **テスト実行**: `pnpm test` で品質保証
5. **ビルド**: `pnpm build` でプロダクションビルド
6. **デプロイ**: Git push で自動デプロイ

## 🔐 セキュリティ

- **環境変数の暗号化**: チーム共有時のセキュリティ確保
- **シークレットローテーション**: 定期的なキー更新
- **HTTPS**: 本番環境の通信暗号化
- **CSP**: Content Security Policy の設定
- **認証トークン**: セキュアな保存と管理
